# 하이로드 수학 해설 제작기 작업 계획서

- 문서 기준일: 2026-04-30

## 프로젝트 목표
- 고화질 시험지 기준으로 문제 추출 및 해설 생성 정확도를 높인다.
- 중고등 교육과정 범위를 벗어나는 용어/기호를 강하게 통제한다.
- 배포 환경(Vercel + Google Drive)에서도 저장 실패 없이 결과물을 안정적으로 생성한다.

- 우측 미리보기 액션 UX 개편(비활성 저장 버튼 제거, 결과 생성 후에만 저장 액션 노출)

- 로컬 `.env.local`에 `OPENAI_API_KEY` 반영으로 GPT 백업 경로 활성화 및 빌드 검증 완료
- 낮은 신호 파일 대응용 번호 기반 완화 추출(`relaxed`) 추가 및 재스모크 테스트(해당 샘플은 여전히 저커버리지 경고)
- AUTONUM 블록 구조 추출기 추가 및 재스모크 수행(해당 샘플은 구조 추출 0건으로 추가 규칙 설계 필요)
- 전문가 토의 기반 작업 규칙 추가(`expert-discussion-workflow.mdc`) 및 ENDNOTE 제외 본문 문단추출 완화 적용(대상 샘플은 여전히 fallback 2문항)
- ENDNOTE 태그 경계 오매치 수정 및 fallback 커버리지 우선 선택 적용(대상 샘플 2문항 유지, 추가 포맷 분석 필요)
- AI 문항 추출 fallback 추가(추출 수 증가 시만 교체) 및 재스모크 수행(대상 샘플 AI 추출 0건)
- 사용자 대표 요청 파일군 전용 프로필(`core-request-set`) 적용(해설 확보 우선 경로 + 진단 노출)
- 전문가 상세코드/프롬프트 리뷰 리스크 수정(재시도 예외 안정화, quickAnswerMode 판정 정합화)
- 원문 구조 디코더 2차 적용(정답표 번호 앵커 기반 세그먼트: 평문 번호 분할 + 문단 AUTONUM 결합) 및 실샘플 재스모크 수행(`questionCount=4`, `coverageRatio=20%`)
- HML 수동 문항 범위 선택 기능 추가(`1-30`, `1,3,5-8` 포맷) 및 API 필터링/응답 진단 연동
- 원문 구조 디코더 3차 1차분 적용(태그 계층 기반 번호-본문-보기 결합) 및 봉명고 샘플 재스모크(`questionCount=2`, `coverageRatio=10%`)
- 원문 구조 디코더 3차 2차분 적용(`ENDNOTE` 보기 기반 복원 추출) 및 대표 샘플 재스모크 통과(봉명고 `questionCount=20`, `coverageRatio=1.0`, 해동고 `questionCount=11`, `coverageRatio=1.0`)
- 모델 백업 장애 가드 적용(Gemini/GPT 모두 실패 시 문항 단위 수동검토로 대체, 전체 배치 연속 처리 유지)
- 대표 샘플 자동 스모크 스크립트 추가(`npm run smoke:hml`) 및 타임아웃 기반 실패 분류(기본 240000ms, 환경변수 `HML_SMOKE_TIMEOUT_MS`)
- 수동 영역지정 보강(구분선 모드 좌표% 입력으로 수동 구분선 추가)
- `smokeFast` 판정 보정(처리 문항 수 기준 커버리지) 및 샘플 선택 실행(`HML_SMOKE_SAMPLES`) 추가
- 영역지정 메인 전환 1차 반영(UI 기본 CTA/표기 수동 중심, HML `mode=manual|auto_assist` 분기, 자동보조 품질 이슈 시 수동 전환 가이드 응답)
- 빠른정답/해설참고 페이지 지정 기능 반영(PDF): 지정 페이지를 필수 작업에서 제외하고 생성 프롬프트 참고 힌트로 전달
- 페이지 지정 UX 보강: `빠른정답 없음`/`해설참고 없음` 체크박스 추가(체크 시 입력 비활성 + 검증 제외)
- 수동 지정/대기열 표기 통일: `영역 박스`/`그림` 라벨을 `문제 박스`/`문제 n`으로 정리
- 편집 가시성/속도 보강: 문제 지정 커서를 고대비로 변경하고, PDF 렌더링에 모듈/문서 캐시를 적용해 편집 진입/페이지 이동 지연을 완화
- 결과물 포맷 정합화: 빠른정답 2열 구성 + 문항 유형별 표기(객관식 번호/단답형 값/서술형 `해설참고`)로 재정의
- 생성 안정성 보강: `문제풀이→빠른정답→해설` 순서 강제 및 해설 미완결(중간 잘림) 응답 재시도 규칙 적용
- 중복 저장 버튼 제거(작업 완료 폴더 수동 저장 삭제, PDF 저장만 노출)
- 빠른정답/해설 참고 페이지 미존재 케이스 체크 보강(빈 입력 정상 처리, 잘못된 페이지 지정 경고)
- 문제 지정 방식 단순화(구분선/세로선 제거, 영역 박스 지정 메인 전환)

## 다음 작업 우선순위(정확도 중심)

   - 문항 하단 조건식 인식률 기준 추가
2. 원문 구조 디코더 품질 보강
   - `ENDNOTE` 복원 경로에서 "보기만 존재" 문항에 대한 생성 프롬프트 안전장치 강화
   - 대표 4개 샘플 전체 재스모크(수동선택 1-30 기준) 수치 고정
3. 교육과정 범위 통제 고도화
   - 금지어 사전 확대(동의어/영문 표기 변형)
   - 감지 결과 UI 경고 및 교체 제안 문구 자동화
4. 운영 검증 자동화
   - `smoke:hml` 결과를 기준으로 타임아웃/실패 원인 분류 리포트 템플릿 고정
   - 모드별 실행(`HML_SMOKE_MODE=manual|auto_assist`) 결과 비교 및 PASS 컷 분리 기록
   - 장시간 샘플(보인고/부산중앙여고) 원인 분리: `smokeFast`로도 시간초과가 나는지 재측정 후 API 단계별 로그 추가
   - `/api/precheck-extraction` + `/api/generate-explanation` 연속 스모크 케이스 정립
   - 실제 시험지 샘플 기반 PASS 기준치(점수/누락 항목) 문서화

## 운영 실행 절차(고정)
1) 로컬 테스트 -> 2) 커밋/푸시 -> 3) Vercel redeploy -> 4) 스모크 테스트 -> 5) docs 3종 업데이트

## 문서 작업 규칙(고정)
- 모든 작업 시작 전에 `docs/plan.md`, `docs/context.md`, `docs/checklist.md`를 먼저 읽고 진행한다.
- 코드/설정/배포/테스트 변경이 발생하면 같은 작업 사이클에서 docs 3종을 반드시 함께 갱신한다.
- 위 규칙은 `.cursor/rules/docs-workflow.mdc`로 상시 적용한다.

## 배포/운영 체크 포인트
- Vercel 환경변수는 반드시 동일 OAuth 세트 사용:
  - `GOOGLE_CLIENT_ID`
  - `GOOGLE_CLIENT_SECRET`
  - `GOOGLE_REFRESH_TOKEN`
- Drive 폴더명 환경변수:
  - `GOOGLE_DRIVE_PARENT_FOLDER_NAME=해설제작`
  - `GOOGLE_DRIVE_EXAMS_FOLDER_NAME=시험지`
  - `GOOGLE_DRIVE_COMPLETED_FOLDER_NAME=작업완료`
