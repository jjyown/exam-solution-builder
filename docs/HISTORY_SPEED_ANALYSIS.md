# “예전엔 빨랐는데 지금은 느리다” — Git·코드 기준 분석

- 문서 기준일: 2026-05-04
- **성격:** 저장소 커밋·코드·운영 환경을 아우르는 **전문가 토의 요약** (절대 원인 단정 아님)

---

## 1. 한 줄 결론

| 관점 | 요지 |
|------|------|
| **저장소 쪽** | “신속”보다 **429 방지·품질 게이트·교차 검증·재시도** 를 키우는 커밋이 **다수** → **같은 1문항도 API 왕복·검증 횟수가 늘 수밖에 없다.** |
| **Google API 쪽** | `gemini-2.0-flash-lite` 등 **구 모델이 신규 키에서 404** 가 나는 사례가 있어, **모델 폴백·재시도** 가 추가 지연·실패감을 만든다. |
| **Cursor 사용 방식** | 예전: 채팅에 그림 첨부(인코딩이 UI 뒤에서 처리) / 지금: **에이전트가 `imageBase64` 를 한 JSON에 실을 때** → **보내는 경로·한도** 가 달라 “전달이 안 된다/느리다” 체감이 생길 수 있음. **이건 Git에 안 잡힌다.** |

---

## 2. Git에서 보이는 “느려질 수밖에 없는” 변화 (발췌)

`git log` 상 해설·배치·API와 맞닿는 커밋들의 **의도**는 대체로 아래와 같다.

| 시기(상대) | 대표 커밋 메시지 | 토의에서의 해석 |
|------------|------------------|-----------------|
| 초기~중기 | `vision precheck`, `quality gate`, `교차검증` | 한 문항에 **LLM 2회+·검증·재시도** 가 붙을 수 있음 → **지연·비용↑, 실패율↓** |
| 429 대응 | `reduce 429 by ... increasing batch **delay**` | 배치는 **의도적으로 느리게** (한도 보호) |
| 이후 | `OpenAI 교차검증 옵션` | `EXPLANATION_CROSS_VERIFY=true` 이면 **Gemini 후 OpenAI** → **최소 2배 왕복** |
| 품질 | `해설 품질 게이트`, `재시도 유도`, `repair` | 스타일/형식 보정 **추가 호출** 가능 |
| 운영 | `MCP·스크립트·문서 추가` | **경로가 늘어남**(학습 비용↑)이지, 단일 호출이 반드시 느려진 것은 아님 |

**합의:** “예전 프로젝트 = 단순 1회 생성”에 가깝다면, **지금 저장소는 의도적으로 파이프라인이 무거워진 상태**라 **평균 응답 시간이 길어진 것이 정상 범주**에 들어간다.

## 3. 코드에 남아 있는 “추가 지연” 요인 (현재 트리)

- **배치 CLI** (`scripts/batch-crops-to-docx.mts`): 문항 사이 **`--delay-ms` 기본 800ms** — 429 완화용.
- **기본 Gemini 후보** (`src/lib/geminiDefaultModels.ts`, MCP도 동일 import): `2.5-flash-lite` → `2.5-flash` 순 시도. ~~`2.0-flash-lite`~~ 는 신규 키에서 404가 나 와 폴백에서 제외.
- **교차 검증** ([models.md](./models.md)): 켜져 있으면 **추가 라운드트립**.

## 4. Git 밖 요인 (분석에 꼭 포함)

| 요인 | 설명 |
|------|------|
| **네트워크·할당량** | 같은 코드라도 시간대·키·쿼터에 따라 체감 속도 상이. |
| **모델 정책 변경** | Google 쪽 모델 가용성·가격·속도는 **커밋 없이** 바뀔 수 있음. |
| **Cursor 에이전트 vs 채팅 첨부** | 동일 MCP라도 **도구 인자 크기 한도** 로 큰 Base64 일괄 전달이 실패하거나 잘리면 “예전엔 됐는데”가 발생할 수 있음. |

## 5. 조치 우선순위 (전문가 합의안)

1. **속도 우선이면:** `.env.local` 에서 **`EXPLANATION_CROSS_VERIFY=false`(또는 미설정)** 확인, 배치 시 **`--delay-ms`** 를 필요 최소로.
2. **모델:** `GEMINI_MODELS_GENERATE_*` 에 **한 개만 쓸 모델**을 명시해 **폴백 시도 자체를 줄인다** (문서: [models.md](./models.md)).
3. **MCP:** 채팅에서 **이미지 첨부 + 짧은 `task`** 가 에이전트 대량 Base64보다 안정적인 경우가 많음 ([CURSOR_MCP_WORKFLOW.md](./CURSOR_MCP_WORKFLOW.md) 상단 Q&A).
4. **품질 우선이면:** 위를 줄이면 다시 빨라지지만 **게이트·검증을 포기한 만큼** 검수 부담이 사람에게 간다 — 목표에 맞게 선택.
