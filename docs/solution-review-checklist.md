# 해설·정답 검토 체크리스트 (전문가 합의용)

자동화(`validateMergedExplanationMarkdown`, `explanationExportGate`)은 **형식·LaTeX 잔존·빠른정답↔해설 구간 형태**를 보조한다. **수학적 정답의 최종 책임은 사람 검토**가 필요하다.

## 1. 데이터 출처 일관성

- **최종 원고**는 `합본_편집용.md` 한 곳을 기준으로 삼는다. DOCX는 항상 이 합본(또는 동일 내용 DB 스냅샷)에서 다시 생성한다.
- 구버전 API 초안(`문항xx_API초안.md`)·스크린샷·이전 PDF만 보고 “아직 안 고쳐졌다”고 판단하지 않는다. **파일 날짜·해시**를 맞춘다.

## 2. 지수함수 + 그래프 (문항 5 유형)

1. **밑의 범위:** $0<a<1$이면 $a^{f(x)}\ge a^{g(x)}\Leftrightarrow f(x)\le g(x)$, $a>1$이면 부등호가 반대다.
2. **그래프에서 $f(x)\le g(x)$:** 교점이 두 개이면, 성립 구간이 교점 **사이**인지 **양쪽 바깥**인지 그림에서 확인한다.  
   - 사이: 보통 $m\le x\le M$  
   - 바깥: 보통 $x\le m$ **또는** $x\ge M$
3. **[빠른 정답] 한 줄**과 **[해설] 마지막 결론**이 같은 **유형**(한 구간 vs 두 구간의 합)인지 눈으로 맞춘다.

## 3. 합본 게이트 실행

- `write-final-docx.mts` 또는 `build-markdown-assembly` 경로에서 구조 검사·내보내기 게이트를 통과시킨다.
- 새 경고: **빠른 정답은 한 구간인데 해설 말미는 「또는」 두 덩어리**처럼 보이면, 위 2번을 재점검한다.

## 4. 자동 검사의 한계

- 같은 형태로 **틀린 숫자**만 적은 경우(예: $-3\le x\le 2$)는 자동으로 잡기 어렵다 → 표준 해·그림 좌표와 대조.
- LLM이 생성한 초안은 **반드시 합본에 편입한 뒤** 게이트를 다시 돌린다.
