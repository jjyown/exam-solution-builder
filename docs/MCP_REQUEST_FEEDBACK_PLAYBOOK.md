# MCP 요청·피드백 플레이북

- 문서 기준일: 2026-05-04
- 목적: MCP 호출 시 오인식·오답을 줄이고, 호출 직후 **다음 요청 개선안**을 매번 남기기 위한 운영 기준.

## 1) 기본 요청 템플릿 (복붙)

```text
아래 이미지(단일 문항 크롭)만 기준으로 풀어라.
출력 형식은 반드시 [정답] 한 줄 + [해설] 한 블록만.
객관식이면 [정답]은 보기 번호(예: ③번)로 쓰고, 계산값만 단독 기재하지 마라.
근호 중첩/곱, 보기 번호, 그래프 축·눈금, 조건 누락 여부를 마지막에 한 번 더 검산해라.
다른 문항 내용이 보이더라도 현재 문항만 출력해라.
```

## 2) 어떤 이미지를 같이 보내야 정확도가 오르는가

| 상황 | 권장 이미지 | 이유 |
|---|---|---|
| 객관식(보기가 작음) | 문제 원본 + 보기 영역 확대 | 보기 번호/식 오인식 방지 |
| 그래프 문제 | 문제 원본 + 그래프만 확대(축·눈금 포함) | 교점/증감/범위 판정 정확도 향상 |
| 도형 문제 | 문제 원본 + 도형만 확대(각/길이 표기) | 조건 누락 방지 |
| 긴 조건 문항 | 상단 조건 + 하단 조건 분할 캡처 | 줄바꿈/부등식 경계 오독 방지 |
| 표/함수값 문제 | 문제 원본 + 표만 확대 | 행·열 값 혼동 방지 |

## 3) 오답 방지용 추가 지시(짧게)

- “근호는 **중첩인지 곱인지** 먼저 식으로 재작성 후 계산.”
- “보기 번호와 계산 결론이 다르면 결론을 다시 계산해서 통일.”
- “그래프는 축·눈금·정의역을 먼저 읽고 식을 세울 것.”
- “최종값은 원조건 대입 검산 후 종료.”

## 4) MCP 호출 뒤 항상 남길 피드백 포맷

```text
## 요청 개선 피드백
1) 다음 호출용 권장 프롬프트: ...
2) 추가 이미지 권장: ... (이유 ...)
3) 오답 방지 체크포인트: ...
4) 자동 개선안: ... (예: OpenAI 교차검증 1회)
```

## 5) 자동 개선 실행 원칙

- 호출 결과에 아래 신호가 있으면 즉시 개선 루프를 건다.
  - 정답/해설 결론 불일치
  - 객관식인데 번호/내용 혼선
  - 근호·분수·지수 구조가 애매
  - 그래프 축/눈금 언급 누락
- 개선 루프 우선순위:
  1. 이미지 보강(보기/그래프 확대)  
  2. 같은 문항 재호출(지시 강화)  
  3. OpenAI 교차검증 재호출
