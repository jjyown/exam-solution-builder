- `01_표준_작업프롬프트.md`
- `02_문항별_검수체크리스트.md`
- `03_실패패턴_교정사전.md`

- `06_v2.1_전문가토의_운영스키마.md` — 전문가 토의 반영 운영안(v2.1), 판정식/SLO/로그 스키마 포함

새 채팅에서도 위 문서를 먼저 읽고 작업하면 결과 편차를 줄일 수 있습니다.

## 권장 실행 순서

1. `00_운영원칙.md` 확인
2. `01_표준_작업프롬프트.md`를 템플릿으로 작업 시작
3. 문항별 작성 후 `02_문항별_검수체크리스트.md` 적용
4. 문제가 발생하면 `03_실패패턴_교정사전.md`로 즉시 교정
5. DOCX 직전 `04_내보내기전_최종게이트.md` 전체 통과 확인
6. 세션 종료 시 `05_세션종합_다음작업_토의록.md`의 **§3 체크리스트**로 보내기·육안 검수를 닫는다

## MCP 사용 팁

- **MCP만 켜 두었다고** 새 채팅에서 자동으로 볼트를 뒤지지는 않습니다. **`.cursorrules`(또는 `.cursor/rules`)**에 “답하기 전에 읽을 파일·순서”를 적어 **트리거**를 거는 것이 핵심입니다. 상세는 `00_운영원칙.md`의 「MCP 연결 vs 규칙」절을 참고하세요.
- Obsidian MCP는 "기억/정책 계층", 코드 게이트는 "실행/검증 계층"으로 분리해 운용합니다.
- OpenAI preflight는 기본 soft 모드(`--preflight-openai`)로 두고, 릴리즈 직전만 strict(`--preflight-openai-strict`)를 권장합니다.
