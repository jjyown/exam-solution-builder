# 00. 운영 원칙

## 목적

- 채팅이 바뀌어도 해설 품질과 형식의 일관성을 유지한다.

## MCP 연결 vs 규칙(트리거) — 전제 고정

| 구분 | 의미 | 비유 |
|------|------|------|
| **MCP 연결** | Cursor에 등록된 MCP(예: 옵시디언·파일시스템)는 새 채팅·재시작 후에도 **백그라운드에서 사용 가능한 권한**으로 남는다. | 열쇠(마스터키)를 계속 소지 |
| **규칙·지시** | 권한만으로는 에이전트가 **질문 없이 자동으로** 볼트/문서를 열지 않는다. “답하기 전에 반드시 이 문서를 읽는다”는 문장은 **`.cursorrules`**, **`.cursor/rules`**, 또는 **사용자가 채팅에 명시**해야 일관되게 실행된다. | 방아쇠(트리거) |

- **완전 자동화**에 가깝게 가려면: `[MCP 접근 권한]` + `[프로젝트 규칙에 읽기 순서·파일명 명시]`를 함께 둔다.
- **본 레포의 권위 문서 경로**(Git에 포함된 “옵시디언용” 세트): `highroad-math-solution/docs/obsidian-mcp/` 의 `00`~`05`. Vault에만 두고 싶다면 **동일 내용을 복제**하거나, 규칙에 **Vault 내 실제 파일명**(예: `해설지_절대규칙.md`)과 **MCP로 읽는 방법**을 명시한다.
- 에이전트는 MCP가 없어도 **Read 도구로 위 경로의 md를 직접 읽는 것**으로 동일 효과를 낼 수 있다(워크스페이스에 클론된 경우).
- **Cursor 워크스페이스 루트**가 `시험지 해설 제작`(상위)이면 상위 `.cursorrules`에 MCP·권위 문서 트리거가 있다. 루트가 **`highroad-math-solution`만**이면 이 폴더의 `.cursorrules`가 적용되도록 **동일 트리거를 복제**해 두었다.

## 고정 원칙

- 문항별로 `[문제]`, `[빠른 정답]`, `[해설]` 순서를 유지한다.
- 객관식 정답은 선택지 번호와 해설 결론이 반드시 일치해야 한다.
- 포기/회피 문장(예: "풀 수 없음", "판독 불가")은 허용하지 않는다.
- 수식은 내보내기 게이트 규칙을 따르고, 필요 시 표현을 단순화한다.
- 도형 파일(`*_fig*`)은 문항 수 집계에서 제외하고 보조 그림으로만 사용한다.

## 의사결정 우선순위

1. 정답 정확성
2. 해설 논리 일관성
3. 내보내기 형식 적합성
4. 표현 가독성

## 예외 처리

- 정답 판단이 모호한 문항은 "보류"로 표시하고 근거를 2개 이상 기록한다.
- OpenAI preflight 실패 시 기본은 soft-fail로 계속 진행하되, 릴리즈 직전에는 strict 모드로 재검증한다.
