# 06. v2.1 전문가 토의 기반 운영 스키마

작성일: 2026-05-05
목적: 기존 v2 파이프라인을 정확성/안정성/재현성 기준으로 강화해 즉시 운영 가능 상태로 고정한다.

## 1) v2.1 전체 흐름

```text
[입력: 크롭 이미지/문항]
  -> (0) 입력 품질 점검
  -> (1) 1차 풀이 생성 (Gemini, 필요 시 OpenAI fallback)
  -> (2) 자동검사 (형식/분량/부정메타)
  -> (3) 문항별 무결성 + 수학 논리 검수
  -> (4) 불합격 즉시 2차 풀이 + 합본 동기화
  -> (5) 최종 게이트 (preflight soft/strict + export gate)
  -> (6) DOCX 생성
  -> (7) 최종 브리핑 + 오류 패턴 환류
```

## 2) 단계별 기준 (운영 고정값)

### (0) 입력 품질 점검

- 문항 경계가 명확하고 본문이 잘리지 않았는가
- 회전/기울기/해상도 문제로 식 판독이 흔들리지 않는가
- 도형 보조 이미지(`*_fig*`)가 본문 문항으로 오집계되지 않았는가

### (1) 1차 풀이 생성 + fallback 트리거

- 기본: Gemini 우선
- fallback: OpenAI
- fallback 트리거
  - 모델 응답 실패/타임아웃
  - `[정답]`/`[해설]` 형식 미준수
  - 결론 누락 또는 자가모순

### (2) 자동검사

- 객관식 정답 형식
- 잘림 의심/분량 과다
- 포기·회피 문장(못 풂, 판독 불가 등)

### (3) 문항별 무결성 + 수학 논리 검수

- `[문제]`, `[빠른 정답]`, `[해설]` 존재
- 정답과 해설 마지막 결론 일치
- 아래 수학 오답 방지 7체크를 통과

## 3) 수학 오답 방지 7체크 (필수)

1. 사칙/지수/로그/삼각 기본 연산 오류 없음
2. 공식/정리 적용 조건 위반 없음
3. 식 전개 과정의 논리 비약 없음
4. 정의역/치역/범위/부호 조건 반영
5. 특수해/경계값/배제조건 누락 없음
6. 문제 요구 출력(값/형식/단위) 충족
7. 객관식은 선택지 번호와 결론이 동일

## 4) 불합격 즉시 2차 풀이 판정식

```text
IF (
  생성 실패 OR
  자동검사 실패 OR
  무결성 실패 OR
  수학 논리 7체크 중 1개 이상 실패
)
THEN
  [검수 불합격] 선언
  -> 해당 문항 2차 풀이 즉시 수행
  -> 문항XX_API초안.md 덮어쓰기
  -> 합본_편집용.md 동기화
```

## 5) 최종 게이트 기준

- 기본 운영: soft preflight
- 마감 전/최종본 직전: strict preflight 재실행
- explanationExportGate 통과 후에만 DOCX 생성

## 6) 운영 SLO (권장)

- (0) 입력 품질: 문항당 1분 이내
- (1) 1차 생성: 문항당 3분 이내
- (2) 자동검사: 문항당 1분 이내
- (3) 논리 검수: 문항당 2분 이내
- (4) 2차 풀이: 문항당 3분 이내
- (5) 최종 게이트: 시험지당 2분 이내

## 7) 최소 로그 스키마 (JSON 키)

## 8) 최종 브리핑 템플릿

- 수정 문항: `n, n, n`
- 불합격 -> 2차 풀이 적용 문항: `n건`
- 남은 리스크: `요약 1~3줄`
- 최종 생성 파일: `경로`
- 반복 오류 패턴: `다음 세션 반영 포인트`

## 9) 2차 전문가 토의 합의 (결함 런북)

주의: 토의 응답 중 파이프라인 범위를 벗어난 항목은 제외하고, 해설 제작 흐름에 직접 연결되는 결함만 채택한다.

### 고위험 결함 6개

1. 문항 초안에서 `[정답]`/`[해설]` 형식 누락
2. 문항 초안 결론과 `[빠른 정답]` 불일치
3. 합본 단계에서 문항 누락/중복/순서 오류
4. 2차 풀이 후 `문항XX_API초안.md`와 `합본_편집용.md` 미동기화
5. preflight 또는 export gate 실패 상태에서 DOCX 생성 강행
6. DOCX 변환 후 수식/레이아웃 깨짐 미검출

### 탐지 규칙 + 즉시 조치

- 결함 1 탐지: 문항 헤더 3종(`[문제]`/`[빠른 정답]`/`[해설]`) 정규식 검사 실패
  - 조치: 해당 문항 즉시 불합격 처리 후 2차 풀이 재작성
- 결함 2 탐지: 해설 마지막 결론과 정답 헤더 비교 불일치
  - 조치: 결론을 기준으로 정답 헤더 동기화 후 재검수
- 결함 3 탐지: 합본 문항 번호 시퀀스 검증(누락/중복/역순)
  - 조치: 합본 롤백 후 문항 단위 재병합
- 결함 4 탐지: 문항 파일 해시와 합본 구간 해시 불일치
  - 조치: 2차 풀이 결과를 합본 동일 구간에 강제 반영
- 결함 5 탐지: 게이트 상태값이 fail인데 DOCX 생성 로그 존재
  - 조치: 생성본 무효 처리, 게이트 통과 후 재생성
- 결함 6 탐지: DOCX 샘플 육안 점검(수식 깨짐/보기 잘림/페이지 넘김 이상)
  - 조치: 원본 md 또는 변환 옵션 교정 후 DOCX 재생성

### 최종 필수 차단 규칙 5개

1. 게이트 fail 상태에서는 DOCX 생성을 금지한다.
2. 수학 논리 7체크 1개라도 실패하면 즉시 2차 풀이로 전환한다.
3. 합본 문항 시퀀스(누락/중복/역순) 오류가 있으면 다음 단계로 진행하지 않는다.
4. 2차 풀이 후 문항 파일-합본 동기화 불일치가 있으면 배포를 차단한다.
5. DOCX 육안 샘플 점검에서 수식 깨짐이 확인되면 최종본으로 승인하지 않는다.

## 10) 자동 연속 진행 프로토콜 (무확인 기본)

원칙: 중요 필수 절차는 사용자 재확인 없이 연속 수행한다. 단, 아래 예외에 해당하면 즉시 중단 후 보고한다.

### 자동 진행 가능 단계 8개

1. 문항 초안 형식 점검
2. 자동검사(형식/분량/부정메타)
3. 무결성 + 수학 논리 7체크
4. 불합격 문항 2차 풀이
5. 문항 파일과 합본 동기화
6. preflight soft 실행
7. preflight strict 실행(마감 전)
8. export gate 통과 확인 후 DOCX 생성/검증

### 반드시 중단 후 보고할 예외 5개

1. 정답 자체가 기존 기준과 충돌해 개념 재판정이 필요한 경우
2. 문항 다수(예: 30% 이상)에서 동시 불합격이 발생한 경우
3. 게이트 실패가 동일 사유로 2회 이상 반복되는 경우
4. DOCX 수식 깨짐이 교정 후에도 재발하는 경우
5. 입력 원본(이미지/문항 파일) 손상으로 풀이 근거가 사라진 경우

### 자동 진행 상태 보고 템플릿

`[진행보고] 단계: <단계명> / 결과: <통과|불합격 n건> / 조치: <즉시 조치 내용> / 다음: <다음 단계>`

## 11) 원클릭 자동화 실행 정책 (final:from-input)

- 기본 실행은 strict gate로 운영한다(구조검사 포함).
- 빠른 산출이 필요한 예외 상황에서만 `--fast`를 사용한다.
- strict 실패 시 즉시 원인 로그를 보고하고, 필요한 문항만 교정 후 재실행한다.
- 자동 선택된 workdir 경로를 항상 로그로 출력해 오선택 리스크를 줄인다.
